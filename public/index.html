NC News API Documentation

This is a simple HTTP REST API designed to act as a news server, in which clients are able to interact with. You can filter articles by existing topics, sort them by: title, when they were published, upvotes or comment count in ascending or descending order. Comments can be retrieved on individual articles, new comments can be posted or deleted by the registered user.

Base URL:

Authentication

This API does not currently require authentication to access public endpoints

Endpoints

    GET /api/topics

Making a get request to this endpoint will return a list of all topics within the database

    GET /api/articles

Making a get request to this endpoint will return all articles, automatically sorted by date in descending order

    GET /api/articles/:article_id

Making a get request to this endpoint will return a single article corresponding to its id number

    PATCH /api/articles/:article_id

Making a patch request to this endpoint updates the votes on a single article corresponding to its article_id

    GET /api/articles/:article_id/comments

Making a get request to this endpoint will return all comments from a single article corresponding to its article_id

    POST /api/articles/:article_id/comments

Making a post request to this endpoint will post a new comment to a single article corresponding to its article_id

    DELETE /api/comments/:comment_id

    Delete request available on /api/comment/:comment_id endpoint. This request will delete the given comment by commment_id

    GET /api/users

Making a get request to this endpoint will return all users

Feature requests

    GET /api/articles

Make a feature request to this endpoint, users can sort articles by article_id, title, vote count, date of creation or comment count ordered by ascending or descending order (where articles are automatically sorted by date of creation in descending order)

Make a feature request to this endpoint, articles can be filtered by the topic specified in the query

    GET /api/articles/:article_id

Make a feature request to this end point, allows the user to view the comment count on single articles
